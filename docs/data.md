# Data Schemas, Lifecycle, and Persistence

This document describes data sources, schemas, lifecycle, and persistence rules for the ai-toolkit.

## Data Sources and Sinks

### Data Sources

| Source | Description | Ingestion Point |
|---|---|---|
| `.env` | Environment configuration | All services at startup |
| `data/openclaw/openclaw.json` | OpenClaw configuration | `openclaw-config-sync` container |
| `data/mcp/servers.txt` | MCP server list | `mcp-gateway` container |
| `data/mcp/registry.json` | MCP server metadata | `mcp-gateway` container |
| `data/rag-input/` | RAG input documents | `rag-ingestion` watch directory |
| `models/ollama/` | Ollama model weights | `ollama` container bind mount |
| `models/comfyui/` | ComfyUI model weights | `comfyui` container bind mount |

### Data Sinks

| Sink | Description | Format |
|---|---|---|
| `data/openclaw/workspace/` | Agent workspace files | Markdown (.md) |
| `data/openclaw/extensions/` | OpenClaw plugins | Docker volume |
| `data/ops-controller/audit.log` | Audit log | JSONL (append-only) |
| `data/qdrant/` | Vector database | Qdrant storage format |
| `data/dashboard/` | Throughput/benchmark data | JSON |
| `data/comfyui-output/` | Generated media | Various (PNG, JPG, MP4, etc.) |
| `data/n8n-data/` | n8n workflow data | n8n storage format |

## Data Schemas

### Audit Log Schema

**Location:** `data/ops-controller/audit.log`

**Format:** JSONL (one JSON object per line, append-only)

```json
{"timestamp":"2026-03-22T10:00:00Z","action":"model_pulled","model":"qwen3:8b","status":"success"}
{"timestamp":"2026-03-22T10:01:00Z","action":"service_started","service":"ollama","status":"success"}
{"timestamp":"2026-03-22T10:02:00Z","action":"config_updated","component":"openclaw","status":"success"}
```

**Fields:**
| Field | Type | Description |
|---|---|---|
| `timestamp` | ISO 8601 | Event timestamp |
| `action` | string | Action type (model_pulled, service_started, config_updated, etc.) |
| `model` | string (optional) | Model name (for model-related actions) |
| `service` | string (optional) | Service name (for service-related actions) |
| `component` | string (optional) | Component name (for config updates) |
| `status` | string | Event status (success, failed, etc.) |

### OpenClaw Config Schema

**Location:** `data/openclaw/openclaw.json`

**Format:** JSON

```json
{
  "models": [
    {
      "name": "qwen3:8b",
      "provider": "ollama"
    }
  ],
  "plugins": [
    {
      "name": "plugin-name",
      "enabled": true
    }
  ],
  "channels": {
    "discord": {
      "enabled": true,
      "token": "discord-bot-token-here"
    },
    "telegram": {
      "enabled": true,
      "token": "telegram-bot-token-here"
    }
  },
  "tools": {
    "web": {
      "search": {
        "enabled": false
      }
    }
  }
}
```

**Note:** This file is auto-generated by `openclaw-config-sync` from `.env` values and should not be edited manually.

### MCP Registry Schema

**Location:** `data/mcp/registry.json`

**Format:** JSON

```json
{
  "servers": ["duckduckgo", "n8n", "tavily", "comfyui"],
  "metadata": {
    "n8n": {
      "allow_clients": ["dashboard", "openclaw"],
      "rate_limit": 100
    }
  }
}
```

**Fields:**
| Field | Type | Description |
|---|---|---|
| `servers` | string[] | List of enabled MCP server names |
| `metadata` | object | Per-server configuration |
| `metadata.<server>.allow_clients` | string[] | Allowed client services |
| `metadata.<server>.rate_limit` | integer | Rate limit for the server |

### RAG Chunk Schema

**Location:** `data/qdrant/` (stored in Qdrant vector DB)

**Format:** Qdrant Point

```json
{
  "id": "unique-chunk-id",
  "vector": [0.1, 0.2, ...],
  "payload": {
    "document_name": "example.txt",
    "chunk_index": 0,
    "content": "The actual text content of this chunk...",
    "chunk_size": 400,
    "chunk_overlap": 50
  }
}
```

**Configuration:** Controlled by `.env` variables:
- `EMBED_MODEL` — Embedding model (default: `nomic-embed-text`)
- `RAG_CHUNK_SIZE` — Chunk size in tokens (default: 400)
- `RAG_CHUNK_OVERLAP` — Chunk overlap in tokens (default: 50)

## Data Lifecycle

### 1. Initialization

**Trigger:** `./ai-toolkit initialize` or `ensure_dirs` script

**Actions:**
- Create `data/` subdirectories
- Create `models/` subdirectories
- Seed OpenClaw workspace files (SOUL.md, AGENTS.md, TOOLS.md, etc.) if missing
- Generate `openclaw.json.example` if not present

**Persistence:** All directories created persist across restarts.

### 2. OpenClaw Workspace Seeding

**Trigger:** `openclaw-ensure-workspace` container on startup

**Actions:**
- Check if workspace files exist in `data/openclaw/workspace/`
- Seed from templates (`SOUL.md.example`, `AGENTS.md.example`, etc.) if missing
- Copy `TOOLS.md.example` to `TOOLS.md` (unless `OPENCLAW_SKIP_TOOLS_MD_UPGRADE=1`)

**Persistence:** Workspace files are bind-mounted and persist.

### 3. Model Pull

**Trigger:** `model-puller` service on first start or via dashboard API

**Actions:**
- Read `MODELS` from environment
- Pull each model via Ollama API
- Store model weights in `models/ollama/`

**Persistence:** Model weights persist in `models/ollama/`.

### 4. ComfyUI Model Pull

**Trigger:** `comfyui-model-puller` service on demand

**Actions:**
- Download LTX-2 models (~60GB) from Hugging Face
- Store in `models/comfyui/`

**Persistence:** Model weights persist in `models/comfyui/`.

### 5. RAG Ingestion

**Trigger:** `rag-ingestion` service watches `data/rag-input/`

**Actions:**
1. Watch `data/rag-input/` for new files
2. Read file content
3. Split into chunks (configurable size/overlap)
4. Embed each chunk using `EMBED_MODEL`
5. Store in Qdrant vector DB (`data/qdrant/`)
6. Remove or archive processed file

**Persistence:**
- Input files: Persist in `data/rag-input/` (can be cleaned manually)
- Vector embeddings: Persist in `data/qdrant/`

### 6. Audit Logging

**Trigger:** `ops-controller` on privileged actions

**Actions:**
- Log action to `data/ops-controller/audit.log`
- Append-only, JSONL format

**Persistence:** Audit log persists and grows over time. Consider periodic archival.

### 7. OpenClaw Memory Updates

**Trigger:** OpenClaw agent during conversations

**Actions:**
- Read current `MEMORY.md`
- Update with new memory entries
- Write back to `data/openclaw/workspace/MEMORY.md`

**Persistence:** MEMORY.md persists and is the key file for agent continuity.

### 8. Config Sync

**Trigger:** `openclaw-config-sync` container on startup

**Actions:**
- Read `.env` values
- Update `openclaw.json` with tokens and settings
- Write to `data/openclaw/openclaw.json`

**Persistence:** Config persists but is regenerated on each startup.

## Data Persistence Rules

### Persistent Data (Bind-Mounted)

| Directory | Purpose | Survives Restart? | Survives Rebuild? |
|---|---|---|---|
| `data/openclaw/workspace/` | Agent workspace | ✅ Yes | ✅ Yes |
| `data/openclaw/` | OpenClaw config | ✅ Yes | ✅ Yes |
| `data/openclaw/extensions/` | OpenClaw plugins (Docker volume) | ✅ Yes | ✅ Yes |
| `models/ollama/` | Ollama models | ✅ Yes | ✅ Yes |
| `models/comfyui/` | ComfyUI models | ✅ Yes | ✅ Yes |
| `data/qdrant/` | Qdrant vector DB | ✅ Yes | ✅ Yes |
| `data/rag-input/` | RAG input files | ✅ Yes | ✅ Yes |
| `data/ops-controller/` | Audit log | ✅ Yes | ✅ Yes |
| `data/mcp/` | MCP config | ✅ Yes | ✅ Yes |
| `data/dashboard/` | Dashboard data | ✅ Yes | ✅ Yes |
| `data/comfyui-output/` | Generated media | ✅ Yes | ✅ Yes |
| `data/n8n-data/` | n8n data | ✅ Yes | ✅ Yes |

### Ephemeral Data (Container-Internal)

| Location | Purpose | Survives Restart? | Survives Rebuild? |
|---|---|---|---|
| `/tmp` (tmpfs) | Ephemeral cache | ❌ No | ❌ No |
| Container layer changes | Read-only filesystem | ❌ No | ❌ No (unless committed) |

## Backup and Recovery

### Critical Files to Backup

1. **OpenClaw Workspace:** `data/openclaw/workspace/` (especially `MEMORY.md`)
2. **OpenClaw Config:** `data/openclaw/openclaw.json`
3. **Model Weights:** `models/ollama/`, `models/comfyui/`
4. **Audit Log:** `data/ops-controller/audit.log`
5. **Qdrant Storage:** `data/qdrant/`

### Backup Strategy

**Option 1: Full Directory Backup**
```bash
# Backup all data
tar -czf ai-toolkit-backup.tar.gz data/ models/
```

**Option 2: Selective Backup**
```bash
# Backup critical data only
tar -czf ai-toolkit-critical-backup.tar.gz \
  data/openclaw/workspace/ \
  data/openclaw/openclaw.json \
  data/ops-controller/audit.log
```

**Option 3: Qdrant Snapshot (RAG)**
```bash
# Qdrant creates snapshots automatically in data/qdrant/snapshots/
ls data/qdrant/snapshots/
```

### Recovery Steps

1. **Stop all services:**
   ```bash
   docker compose down
   ```

2. **Restore data from backup:**
   ```bash
   tar -xzf ai-toolkit-backup.tar.gz
   ```

3. **Restart services:**
   ```bash
   docker compose up -d
   ```

## Data Migration

### Moving Data Directory

To move `data/` to a different location:

1. **Update `.env`:**
   ```
   DATA_PATH=/new/path/to/data
   ```

2. **Ensure new directory exists:**
   ```bash
   mkdir -p /new/path/to/data
   ```

3. **Copy existing data:**
   ```bash
   cp -r data/* /new/path/to/data/
   ```

4. **Restart services:**
   ```bash
   docker compose down
   docker compose up -d
   ```

## Data Cleanup

### Periodic Cleanup Tasks

| Data | Action | Frequency |
|---|---|---|
| `data/ops-controller/audit.log` | Archive old entries | Monthly |
| `data/rag-input/` | Remove processed files | As needed |
| `data/comfyui-output/` | Remove old outputs | As needed |
| `models/ollama/` | Remove unused models | Quarterly |

### Cleanup Commands

**Audit Log:**
```bash
# Archive old audit log
mv data/ops-controller/audit.log data/ops-controller/audit.log.$(date +%Y%m%d)
```

**RAG Input:**
```bash
# Remove processed files
rm data/rag-input/*.processed
```

**Ollama Models:**
```bash
# List models
docker compose exec ollama ollama list

# Remove unused model
docker compose exec ollama ollama rm <model-name>
```
